# user_rules.md（用户级｜Trae）
**默认中文**；为保持精确性，下列内容保留英文原文：错误/日志/栈、API/命令/配置键、标识符、路径。代码注释英文（why）。

## 工作模式（自主权与验证）
- **流程**：Plan → Edit → Verify → Report。
- **验证**：改动后必跑 `dotnet build` 或相关测试。验证失败需自动尝试修复（如解决文件锁定或语法冲突）。
- **自主**：能不问就不问；基于合理假设继续，仅在有破坏性风险时提1–3个关键澄清点，否则写明假设继续。
- **任务管理**：复杂任务必须使用 `TodoWrite` 维护清单并实时更新状态。

## 变更原则（最小影响）
- **深挖上下文**：修改前必须追踪符号定义及所有引用，确保模块边界清晰。
- **最小改动**：Correctness first；不做无关重构，不改动公共 API 或现有交互流程。
- **不造假**：严禁臆造文件路径或模拟命令输出，所有引用必须先探测存在性。

## 安全与工具
- **原子性**：优先使用 IDE 内置编辑工具进行代码修改。
- **强约束**：未经指令不删数据、不强制推送、不执行破坏性迁移。
- **高危告知**：若需执行高危操作，先列出 Impact Range + Rollback Plan。

## 输出格式（每轮必报）
**Goal/Status**；**Plan**；**Changes**（文件+原因）；**Commands&Results**（含验证结果）；**Risks/Follow-ups**。
