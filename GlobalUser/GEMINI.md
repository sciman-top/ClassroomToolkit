## 1. 阅读指引（必读）
- 本文件仅放跨项目稳定规则，不放仓库特定实现细节。
- 三层协同模型：共性基线（全局）+ 平台差异（全局）+ 项目差异（项目级）。
- 默认中文沟通，必要时保留英文术语、命令、日志与报错原文。
- 规则冲突处理：平台加载语义 + 就近原则 + 项目优先，并在回复中说明采用依据。

## A. 共性基线（全局）
### A.1 语言与沟通
- 对用户回复默认中文，除非用户明确要求英文。
- 代码、命令、文件名、日志保持英文原样。
- 发现平台输出切到英文且用户未要求时，立即切回中文继续。

### A.2 交互与输出
- 默认先给目标、边界、执行路径；用户明确要求直做时可直接执行。
- 简单任务使用 Result + Evidence；复杂任务使用 Goal/Plan/Changes/Verification/Risks。
- 存在关键歧义时先澄清最小必要假设。

### A.3 变更与验证
- 仅修改需求相关内容，避免无关重排与样式噪音。
- 先定位再改动；改动后执行最小相关验证。
- 无法验证时，必须说明未验证原因、风险与回滚点。

### A.4 依赖与安全
- 新增或升级依赖前先说明必要性与影响面。
- 高风险操作（例如 `rm -rf`、`git reset --hard`、批量覆盖、破坏性数据变更）必须先提示影响。
- 严禁泄露密钥、令牌、隐私数据；必要时脱敏。

### A.5 提交与评估
- 仅在用户明确要求时执行 `git commit`。
- 评估类任务统一给出 0-100 分、主要缺口、可落地改进动作。
- 协同评分口径固定为五维：覆盖度、边界清晰、执行性、冗余控制、跨平台一致性（各 20 分）。

### A.6 三层边界与协同合同
- 全局层职责：稳定共性基线、平台加载语义、诊断范式、维护清单。
- 项目层职责：仓库结构、领域规则、构建测试命令、风险降级、交付约束。
- 两层关系：互补不重叠；全局不替代项目细则，项目不复制全局平台说明。

### A.7 终态最佳实践导向（新建/重构/扩展）
- 新建项目与重构必须以社区最佳实践终态为目标，不以临时可用为终点。
- 架构优先模块化、结构化、规范化，保证长期维护与可演进。
- 扩展、新增、修复时必须先判定归宿层，禁止把运行时编排和业务规则乱堆到热点文件。
- 若现状与终态存在差距，按目标架构持续收口；禁止继续制造新的结构债务。

## B. Gemini 平台差异（全局，基于官方文档）
### B.1 加载与作用域
- 默认上下文文件名为 `GEMINI.md`，可在配置中调整。
- 全局与项目规则会叠加；子目录规则形成更细粒度约束。
- `.geminiignore` 与 `.gitignore` 会影响扫描范围。

### B.2 配置与上限
- 常见规则发现上限与上下文发现参数以官方文档为准。
- memory 相关命令用于查看、刷新、补充上下文。
- 配置试验后需恢复基线并记录差异。

### B.3 诊断与命令
- 通过 memory 命令核对当前已加载上下文与生效范围。
- 若引入临时记忆或临时上下文，任务结束后应删除并刷新。

### B.4 时效性提示
- 若本文件与官方文档冲突，以官方文档为准，并在回复中标注差异。

## C. 项目差异（全局模板）
- 项目级文件必须自包含，并显式引用对应 GlobalUser 文件。
- 项目级文件必须包含：项目概况、模块边界、数据与隐私、构建测试发布、UI/交互、风险与降级、验证与回滚。
- 项目级 B 节只写平台在本仓库的操作差异；A/C/D 以仓库规则为主。
- 项目级应提供“仅项目/仅全局/两层联动”判定矩阵。

## D. 维护校验清单（全局）
说明：维护清单为附录，不属于三层结构正文。
- 保持 1/A/B/C/D 完整结构与编号一致。
- GlobalUser 三文件 A/C/D 必须一致，仅 B 节允许平台差异。
- 项目三文件 A/C/D 必须一致，仅 B 节允许平台差异。
- 复核六文件引用链：GlobalUser -> 项目级对应文件，语义一致且无冲突。
- 复核三层边界：不重叠、不缺失、可执行。
- 版本号、日期、变更记录需同步更新。

## Execution Contract For Long Refactors
- Continue until done.
- Do not pause for progress reports.
- After each pass, continue to next actionable task.
- On local failure, self-repair first.
- Ask only for true blockers or irreversible risky actions.
- Use non-interactive commands only.
- 用户沟通默认中文；中途仅做最小必要进度播报。
