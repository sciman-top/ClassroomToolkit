# ADR-005: 架构守卫与 CI 阻断策略

- 日期: 2026-02-24
- 最后更新: 2026-03-10
- 状态: Accepted

## 决策

- 使用 `ArchitectureDependencyTests` 作为架构守卫。
- 对历史遗留依赖采用白名单；禁止新增违规。
- 守卫失败阻断合并。
- 白名单只能随边界收口而缩小，不允许为了临时通过而扩张。

## 当前守卫重点

- App 层直接依赖 `ClassroomToolkit.Interop` 的文件数必须继续下降。
- Windowing Adapter / Executor 是允许的主要边界；普通 UI / 场景文件不是新增白名单候选。
- 文档口径与守卫口径必须一致：主方案、主进度、handover 中写明的边界，应与守卫保持同步。

## 说明

当前仍存在历史遗留直连，但这些属于待收口对象，不是目标架构的许可模式。
